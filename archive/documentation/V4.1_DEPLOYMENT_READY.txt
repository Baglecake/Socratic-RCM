**THE CRITICAL ISSUE:**
v4 incorrectly placed agent behavioral heuristics (If/Then and Rival) in
Section 2 (Agent Prompts). This was pedagogically wrong.

**THE CORRECTION:**
Agent behavioral heuristics belong in Section 3 (Round Custom Instructions),
not in agent prompts.

**WHY THIS MATTERS:**
- Round instructions set CONTEXT-SPECIFIC behavioral rules
- Agent prompts define GENERAL agent characteristics
- Same agent can behave differently in different rounds
- This is pedagogically sound: context governs behavior, not identity

**THE FIX:**
Split [Agent Descriptor] into TWO distinct concepts:

1. **[Agent Persona]** (REQUIRED)
   - 2-3 sentence behavioral summary
   - Describes HOW agent generally behaves and makes decisions
   - Goes in Section 2 (agent system prompt)

2. **[Agent Behaviors]** (OPTIONAL)
   - Behavioral heuristics for specific rounds
   - If/Then and Rival formats OR custom rules (1-2 sentences each)
   - Goes in Section 3 (round custom instructions)

**Common Questions:**

Q: "Do I have to use Agent Behaviors?"
A: No! They're optional but recommended. If your theory doesn't
   need specific heuristics, you can skip them or use custom rules.

Q: "Can I use custom behavioral rules instead of If/Then/Rival?"
A: Yes! Describe your behavioral dynamics in 1-2 sentences each.

Q: "Where do Agent Behaviors go?"
A: In Section 3 (Round Custom Instructions), NOT in Section 2
   (Agent Prompts).

Q: "What's the difference between Persona and Behaviors?"
A: Persona = general behavioral summary (who the agent is).
   Behaviors = context-specific rules (how they act in this round).
